<h1 id="anomaly-detection-in-storage-using-an-auto-encoder">Anomaly Detection in Storage Using an Auto Encoder</h1>
<h2 id="assumptions-and-background">Assumptions and background</h2>
<p>In systems across the user space we define the set of vmstores to be a unique storage system defined by a user. In each of the arrays there exist some set of vm’s that are used by the clients. We define the behavior of a single machine based on a set of features, the three main ones being file system latency, time of day and throughput. Each of these vm’s are assumed to come from some probability distribution of behaviors, such that one vm differs in behavior from another, however for a single vm the behavior of that machine contains less noise from its day to day use. We aggregate these vm behaviors over a vmstore, for model complexity reasons.</p>
<p>Previous attempts have been utilized by different work such as <a href="http://www.cs.technion.ac.il/~mgabel/papers/MSc%20Thesis%20-%20Final.pdf">Unsupervised Anomaly Detection in Large Datacenters</a> however the paper makes an assumption across the machines <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi><mo>∈</mo><mi>M</mi></mrow><annotation encoding="application/x-tex">m \in M</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.5782em; In our case we assumed we would have no access to that (considering there are millions of vm’s over the whole dataset, and several thousand vmstores that are aggregated, tasking to figure that out would be a lot of resources and likely not worth our time). In order to mitigate pestering our clients for all ~5000 vmstores, we tried a few other approaches. This approach the team belives will help minimize the resources needed to create an effective alert system.</p>
<p>Taking some direction from <a href="https://arxiv.org/pdf/1708.03665.pdf">Googles paper on Anomaly detection in Time Series</a>, we approached it with an Auto Encoder as well as a Long Short Term Memory Network (<strong>LSTM</strong>). A sample of the results will be shown below on this notebook. For now we will specifically focusing on the autoencoder, at a very high level.</p>

<h3 id="toy-problem">Toy Problem</h3>

<p>1: The day to day of the system it is applied to has a lot of noise<br>
